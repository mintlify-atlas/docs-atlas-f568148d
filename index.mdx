## How it works

OpenTelemetry Python Contrib wraps popular libraries at import time — intercepting calls to generate spans, metrics, and log records that flow to any OpenTelemetry-compatible backend (Jaeger, Zipkin, OTLP collectors, cloud providers, and more).

<Steps>
  <Step title="Install the instrumentation packages">
    Install `opentelemetry-distro` (which includes the `opentelemetry-instrument` command) plus any per-library packages you need.

    ```bash
    pip install opentelemetry-distro[otlp]
    opentelemetry-bootstrap -a install
    ```
  </Step>
  <Step title="Run your app with auto-instrumentation">
    Prefix your normal start command with `opentelemetry-instrument` to enable zero-code tracing.

    ```bash
    opentelemetry-instrument python app.py
    ```
  </Step>
  <Step title="View your traces">
    Telemetry is exported to the configured backend. Use the `console` exporter to verify traces locally during development.

    ```bash
    opentelemetry-instrument --traces_exporter console python app.py
    ```
  </Step>
</Steps>
