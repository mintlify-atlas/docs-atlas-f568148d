---
title: "Quickstart"
description: "Go from zero to running instrumented Python code in three steps."
---

This guide instruments a minimal Flask application, exports traces to the console, and shows you what a span looks like. The same steps apply to any Python application — swap Flask for whichever framework you use.

## Prerequisites

- Python 3.9 or later
- pip

<Steps>
  <Step title="Install packages and auto-detect instrumentation">
    Install `opentelemetry-distro` with the OTLP exporter extra. This pulls in the SDK, the OTLP exporter, and the `opentelemetry-instrument` / `opentelemetry-bootstrap` CLI tools.

    ```bash
    pip install "opentelemetry-distro[otlp]" flask
    ```

    Then run `opentelemetry-bootstrap` to scan your environment and install the matching instrumentation packages:

    ```bash
    opentelemetry-bootstrap -a install
    ```

    `opentelemetry-bootstrap` inspects the packages installed in the active Python environment and installs the corresponding `opentelemetry-instrumentation-*` packages. Because you installed `flask` above, it will install `opentelemetry-instrumentation-flask` (and any other packages it detects).

    To preview what it would install without making changes, omit the `-a install` flag:

    ```bash
    opentelemetry-bootstrap
    ```

    This prints the list of instrumentation packages to stdout, which you can append to a `requirements.txt` file.
  </Step>

  <Step title="Run your app with opentelemetry-instrument">
    Create a minimal Flask application:

    ```python app.py
    from flask import Flask

    app = Flask(__name__)

    @app.route("/")
    def hello():
        return "Hello, world!"

    if __name__ == "__main__":
        app.run(port=5000)
    ```

    Start it with `opentelemetry-instrument`, directing traces to the console so you can see them immediately:

    ```bash
    opentelemetry-instrument --traces_exporter console python app.py
    ```

    Make a request in another terminal:

    ```bash
    curl http://localhost:5000/
    ```

    You will see a JSON span printed to the terminal where the app is running. It looks roughly like this:

    ```json
    {
        "name": "GET /",
        "context": {
            "trace_id": "0x5b8aa5a2d2c872e8321cf37308d69df2",
            "span_id": "0x051581bf3cb55c13",
            "trace_state": "[]"
        },
        "kind": "SpanKind.SERVER",
        "parent_id": null,
        "start_time": "2024-01-15T10:00:00.000000Z",
        "end_time": "2024-01-15T10:00:00.005000Z",
        "status": {
            "status_code": "UNSET"
        },
        "attributes": {
            "http.method": "GET",
            "http.server_name": "127.0.0.1",
            "http.scheme": "http",
            "net.host.port": 5000,
            "http.host": "localhost:5000",
            "http.target": "/",
            "net.peer.ip": "127.0.0.1",
            "http.user_agent": "curl/7.88.1",
            "http.flavor": "1.1",
            "http.route": "/",
            "http.status_code": 200

            "attributes": {
                "telemetry.sdk.language": "python",

            }

    ```

    <Tip>
      Set `OTEL_SERVICE_NAME` to give your service a meaningful name in the span's `resource.attributes`:

      ```bash
      OTEL_SERVICE_NAME=my-flask-app opentelemetry-instrument --traces_exporter console python app.py
      ```
    </Tip>
  </Step>

  <Step title="(Optional) Add programmatic instrumentation">
    The CLI handles most cases, but you can also instrument Flask explicitly in code. This is useful when you need request or response hooks, or when you cannot use the CLI.

    Install the Flask instrumentation package directly if you haven't already:

    ```bash
    pip install opentelemetry-instrumentation-flask
    ```

    Then call `FlaskInstrumentor().instrument_app(app)` before the first request is served:

    ```python app.py
    from flask import Flask
    from opentelemetry.instrumentation.flask import FlaskInstrumentor

    app = Flask(__name__)

    # Instrument the app before any routes are registered or requests are handled
    FlaskInstrumentor().instrument_app(app)

    @app.route("/")
    def hello():
        return "Hello, world!"

    if __name__ == "__main__":
        app.run(port=5000)
    ```

    <Note>
      When using programmatic instrumentation, you are responsible for configuring the SDK (TracerProvider, exporter, etc.) separately. The `opentelemetry-distro` package provides a `configure()` helper, or you can configure the SDK manually using `opentelemetry-sdk`.
    </Note>

    Run it the same way, with or without the CLI wrapper:

    ```bash
    # With CLI (CLI sets up the SDK automatically)
    opentelemetry-instrument --traces_exporter console python app.py

    # Without CLI (you must configure the SDK yourself before running)
    python app.py
    ```
  </Step>
</Steps>
