title: "Database Instrumentation"
description: "Automatically instrument SQL and NoSQL database clients in Python with OpenTelemetry."

OpenTelemetry Python Contrib provides instrumentation for a wide range of database drivers and ORMs. Each package creates spans for database operations and records standard semantic convention attributes such as `db.system`, `db.statement`, and `db.name`.

## Quick Start

<Tabs>
  <Tab title="SQLAlchemy">
    <Steps>
      <Step title="Install the package">
        ```bash
        pip install opentelemetry-instrumentation-sqlalchemy
        ```
      </Step>
      <Step title="Instrument a synchronous engine">
        Pass your engine instance to `SQLAlchemyInstrumentor`. All SQL operations executed through this engine will produce spans.

        ```python
        from sqlalchemy import create_engine
        from opentelemetry.instrumentation.sqlalchemy import SQLAlchemyInstrumentor

        engine = create_engine("postgresql+psycopg2://user:pass@localhost/mydb")
        SQLAlchemyInstrumentor().instrument(engine=engine)

        with engine.connect() as conn:
            result = conn.execute("SELECT 1")
        ```
      </Step>
      <Step title="Instrument an async engine">
        For async SQLAlchemy, pass the underlying sync engine:

        ```python
        from sqlalchemy.ext.asyncio import create_async_engine
        from opentelemetry.instrumentation.sqlalchemy import SQLAlchemyInstrumentor

        engine = create_async_engine("postgresql+asyncpg://user:pass@localhost/mydb")
        SQLAlchemyInstrumentor().instrument(engine=engine.sync_engine)
        ```
      </Step>
      <Step title="Optional: enable SQLCommenter">
        SQLCommenter appends trace context as a comment to outgoing SQL, enabling correlation between database logs and traces:

        ```python
        SQLAlchemyInstrumentor().instrument(
            engine=engine,
            enable_commenter=True,
        )
        # Resulting query: SELECT 1 /*traceparent='00-abc...-01'*/
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Redis">
    <Steps>
      <Step title="Install the package">
        ```bash
        pip install opentelemetry-instrumentation-redis
        ```
      </Step>
      <Step title="Instrument the Redis client">
        `RedisInstrumentor` patches the `redis` library globally so that all client instances are traced.

        ```python
        import redis
        from opentelemetry.instrumentation.redis import RedisInstrumentor

        RedisInstrumentor().instrument()

        client = redis.Redis(host="localhost", port=6379)
        client.set("key", "value")
        value = client.get("key")
        ```
      </Step>
    </Steps>
  </Tab>
</Tabs>

## DB Span Attributes

Spans produced by database instrumentation packages follow the [OpenTelemetry Database Semantic Conventions](https://opentelemetry.io/docs/specs/semconv/database/). Common attributes include:

| Attribute | Description | Example |
|-----------|-------------|---------|
| `db.system` | Database system identifier | `postgresql`, `redis`, `mongodb` |
| `db.statement` | SQL or query statement | `SELECT * FROM users WHERE id = ?` |
| `db.name` | Name of the database | `mydb` |
| `db.operation` | Operation name | `SELECT`, `INSERT`, `GET` |
| `server.address` | Database host | `localhost` |
| `server.port` | Database port | `5432` |

<Note>
  `db.statement` capture may be disabled by default in some instrumentations for security reasons. Check the individual reference page for configuration options.
</Note>
