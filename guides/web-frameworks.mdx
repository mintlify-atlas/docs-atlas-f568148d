---
title: "Web Framework Instrumentation"
description: "Automatically instrument Flask, Django, FastAPI, and other Python web frameworks with OpenTelemetry."
---

OpenTelemetry Python Contrib provides instrumentation packages for the most widely used Python web frameworks. Each package traces incoming HTTP requests, capturing span attributes such as HTTP method, URL, route, and status code.

## Quick Start

<Tabs>
  <Tab title="Flask">
    <Steps>
      <Step title="Install the package">
        ```bash
        pip install opentelemetry-instrumentation-flask
        ```
      </Step>
      <Step title="Instrument your application">
        Flask instrumentation builds on the OpenTelemetry WSGI middleware and patches the Flask application at startup.

        ```python
        from opentelemetry.instrumentation.flask import FlaskInstrumentor
        from flask import Flask

        app = Flask(__name__)
        FlaskInstrumentor().instrument_app(app)

        @app.route("/")
        def index():
            return "Hello, World!"
        ```

        Alternatively, use the global instrumentor before creating the app:

        ```python
        from opentelemetry.instrumentation.flask import FlaskInstrumentor

        FlaskInstrumentor().instrument()

        from flask import Flask
        app = Flask(__name__)
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="FastAPI">
    <Steps>
      <Step title="Install the package">
        ```bash
        pip install opentelemetry-instrumentation-fastapi
        ```
      </Step>
      <Step title="Instrument your application">
        FastAPI instrumentation wraps the ASGI application and traces all incoming HTTP requests automatically.

        ```python
        from opentelemetry.instrumentation.fastapi import FastAPIInstrumentor
        from fastapi import FastAPI

        app = FastAPI()
        FastAPIInstrumentor.instrument_app(app)

        @app.get("/")
        async def root():
            return {"message": "Hello, World!"}
        ```

        You can also use auto-instrumentation via the `opentelemetry-instrument` CLI:

        ```bash
        opentelemetry-instrument uvicorn main:app
        ```
      </Step>
    </Steps>
  </Tab>
  <Tab title="Django">
    <Steps>
      <Step title="Install the package">
        ```bash
        pip install opentelemetry-instrumentation-django
        ```
      </Step>
      <Step title="Instrument your application">
        Django instrumentation traces all requests routed through Django's middleware stack.

        ```python
        from opentelemetry.instrumentation.django import DjangoInstrumentor

        DjangoInstrumentor().instrument()
        ```

        Place this call in your `manage.py` or `wsgi.py` / `asgi.py` entry point, before the Django application is loaded. Auto-instrumentation is also supported via the `opentelemetry-instrument` CLI.
      </Step>
    </Steps>
  </Tab>
</Tabs>

## ASGI and WSGI Middleware

FastAPI, Starlette, and other ASGI frameworks are instrumented through `opentelemetry-instrumentation-asgi`, which provides an `OpenTelemetryMiddleware` compatible with any ASGI application. Similarly, Flask, Django (in WSGI mode), Pyramid, and Falcon are instrumented through `opentelemetry-instrumentation-wsgi`.

Framework-specific packages (e.g. `opentelemetry-instrumentation-fastapi`) layer additional logic on top of the underlying ASGI/WSGI middleware — for example, extracting route templates for cleaner span names. You can use the ASGI/WSGI middleware packages directly if your framework is not listed above:

```python
# Generic ASGI middleware
from opentelemetry.instrumentation.asgi import OpenTelemetryMiddleware

app = OpenTelemetryMiddleware(your_asgi_app)
```

```python
# Generic WSGI middleware
from opentelemetry.instrumentation.wsgi import OpenTelemetryMiddleware

app = OpenTelemetryMiddleware(your_wsgi_app)
```
