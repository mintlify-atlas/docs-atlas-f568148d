---
title: "Tornado"
description: "OpenTelemetry instrumentation for Tornado — traces HTTP requests and responses automatically."
---

Traces web requests in Tornado applications by patching `tornado.web.RequestHandler` and `tornado.httpclient.AsyncHTTPClient` to automatically instrument both incoming server requests and outgoing client requests.

## Installation

```bash
pip install opentelemetry-instrumentation-tornado
```

## Usage

### Auto-instrumentation

`opentelemetry-instrument` handles this automatically.

```bash
opentelemetry-instrument python myapp.py
```

### Manual instrumentation

Call `TornadoInstrumentor().instrument()` before defining your application.

```python
import tornado.web
import tornado.ioloop
from opentelemetry.instrumentation.tornado import TornadoInstrumentor

TornadoInstrumentor().instrument()

class MainHandler(tornado.web.RequestHandler):
    def get(self):
        self.write("Hello, world")

app = tornado.web.Application([(r"/", MainHandler)])
app.listen(8080)
tornado.ioloop.IOLoop.current().start()
```

## Configuration

<ParamField path="server_request_hook" type="callable">
  Called for each incoming request to the Tornado web server. Receives `(span, handler)` where `handler` is a `tornado.web.RequestHandler` instance.
</ParamField>

<ParamField path="client_request_hook" type="callable">
  Called just before an outgoing request via `AsyncHTTPClient.fetch`. Receives `(span, request)` where `request` is a `tornado.httpclient.HTTPRequest`.
</ParamField>

<ParamField path="client_response_hook" type="callable">
  Called after an outgoing `AsyncHTTPClient.fetch` request completes. Receives `(span, future)` where `future` is a `Future[tornado.httpclient.HTTPResponse]`.
</ParamField>

<ParamField path="tracer_provider" type="TracerProvider">
  Optional tracer provider. Defaults to the globally configured provider.
</ParamField>

<ParamField path="meter_provider" type="MeterProvider">
  Optional meter provider. Defaults to the globally configured provider.
</ParamField>

### Environment variables

| Variable | Description |
|---|---|
| `OTEL_PYTHON_TORNADO_EXCLUDED_URLS` | Comma-delimited list of path regexes to exclude from tracing. |
| `OTEL_PYTHON_TORNADO_TRACED_REQUEST_ATTRS` | Comma-delimited list of `HTTPRequest` attributes to capture as span attributes (e.g. `uri,query`). |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_REQUEST` | Request header names (or regexes) to capture as span attributes. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_RESPONSE` | Response header names (or regexes) to capture as span attributes. |

### Hooks example

```python
from opentelemetry.instrumentation.tornado import TornadoInstrumentor

def server_request_hook(span, handler):
    if span and span.is_recording():
        span.set_attribute("tornado.handler", type(handler).__name__)

def client_request_hook(span, request):
    if span and span.is_recording():
        span.set_attribute("http.request.url", request.url)

def client_response_hook(span, future):
    pass

TornadoInstrumentor().instrument(
    server_request_hook=server_request_hook,
    client_request_hook=client_request_hook,
    client_response_hook=client_response_hook,
)
```

## Spans produced

**Server spans** (kind `SERVER`) are created for each incoming HTTP request handled by a `RequestHandler`. The span name is `{METHOD} {path}`. The `tornado.handler` attribute is set to the fully qualified handler class name.

**Client spans** (kind `CLIENT`) are created for each outgoing request made via `tornado.httpclient.AsyncHTTPClient.fetch`.

Standard attributes include:

- `http.method` / `http.request.method`
- `http.scheme`
- `http.host`
- `http.target`
- `http.status_code` / `http.response.status_code`
- `http.flavor`
- `tornado.handler`

## Supported versions

Requires `tornado >= 5.1.1`.
