<Warning>
This package implements GenAI semantic conventions that are currently in **development** status. The API surface, span attributes, and metric names may change in future releases as the conventions evolve.
</Warning>

## Installation

```bash
pip install opentelemetry-instrumentation-openai-v2
```

<Note>
The PyPI package name is `opentelemetry-instrumentation-openai-v2`. The import path is `opentelemetry.instrumentation.openai_v2`.
</Note>

## Usage

### Auto-instrumentation

Use the `opentelemetry-instrument` CLI to instrument without code changes:

```bash
opentelemetry-instrument python app.py
```

### Manual instrumentation

Call `OpenAIInstrumentor().instrument()` before creating any OpenAI clients. Tracing covers chat completions and embeddings.

```python
from openai import OpenAI
from opentelemetry.instrumentation.openai_v2 import OpenAIInstrumentor

OpenAIInstrumentor().instrument()

client = OpenAI()

# Traced chat completion
response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {"role": "user", "content": "Write a short poem on OpenTelemetry."},
    ],
)
print(response.choices[0].message.content)

# Traced embeddings
embedding_response = client.embeddings.create(
    model="text-embedding-3-small",
    input="Generate vector embeddings for this text",
)
```

To stop instrumentation:

```python
OpenAIInstrumentor().uninstrument()
```

### OpenAI-compatible platforms

This instrumentation works with any platform that uses the OpenAI SDK. The `gen_ai.system` span attribute identifies the backend:

| Platform | `gen_ai.system` |
|---|---|
| Azure OpenAI | `azure.ai.openai` |
| Gemini | `gemini` |
| Perplexity | `perplexity` |
| xAI | `xai` |
| DeepSeek | `deepseek` |
| Groq | `groq` |
| MistralAI | `mistral_ai` |

## Configuration

<ParamField path="tracer_provider" type="TracerProvider">
  Custom `TracerProvider` to use. Defaults to the global tracer provider.
</ParamField>

<ParamField path="meter_provider" type="MeterProvider">
  Custom `MeterProvider` to use for recording token usage and operation duration metrics. Defaults to the global meter provider.
</ParamField>

<ParamField path="logger_provider" type="LoggerProvider">
  Custom `LoggerProvider` for emitting log events (prompts and completions). Defaults to the global logger provider.
</ParamField>

### Environment variables

| Variable | Description |
|---|---|
| `OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT` | Controls prompt and completion capture. See values below. |
| `OTEL_SEMCONV_STABILITY_OPT_IN` | Set to `gen_ai_latest_experimental` (or append it) to enable the latest experimental semantic convention features. |

**`OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT` values:**

| Value | Behavior |
|---|---|
| `true` | Legacy mode. Captures content on `gen_ai.{role}.message` and `gen_ai.choice` events when experimental mode is **not** enabled. |
| `span_only` | Captures content on span attributes (experimental mode enabled). |
| `event_only` | Captures content on event attributes (experimental mode enabled). |
| `span_and_event` | Captures content on both span and event attributes (experimental mode enabled). |

Message content is **not captured by default** to avoid recording sensitive data.

## Spans produced

`OpenAIInstrumentor` creates one span per API operation (e.g., `chat.completions.create`, `embeddings.create`).

| Attribute | Description |
|---|---|
| `gen_ai.system` | AI provider identifier (e.g., `"openai"`, `"azure.ai.openai"`) |
| `gen_ai.operation.name` | Operation type, e.g., `"chat"` or `"embeddings"` |
| `gen_ai.request.model` | Model name requested |
| `gen_ai.response.model` | Model name returned in the response |
| `gen_ai.usage.input_tokens` | Number of prompt tokens consumed |
| `gen_ai.usage.output_tokens` | Number of completion tokens produced |
| `server.address` | OpenAI API hostname |
| `server.port` | OpenAI API port |

## Metrics produced

| Metric | Type | Unit | Description |
|---|---|---|---|
| `gen_ai.client.operation.duration` | Histogram | `s` | Duration of the GenAI API operation |
| `gen_ai.client.token.usage` | Histogram | `{token}` | Number of input and output tokens per operation |

## Supported versions

Requires `openai >= 1.26.0`.
