---
title: "Flask"
description: "OpenTelemetry instrumentation for Flask — traces HTTP requests and responses automatically."
---

Builds on the OpenTelemetry WSGI middleware to trace web requests in Flask applications, using the Flask URL rule pattern as the span name and setting the `http.route` attribute.

## Installation

```bash
pip install opentelemetry-instrumentation-flask
```

## Usage

### Auto-instrumentation

`opentelemetry-instrument` handles this automatically. No code changes required.

```bash
opentelemetry-instrument flask run
```

### Manual instrumentation

<Tabs>
  <Tab title="Instrument existing app">
    ```python
    from flask import Flask
    from opentelemetry.instrumentation.flask import FlaskInstrumentor

    app = Flask(__name__)

    FlaskInstrumentor().instrument_app(app)

    @app.route("/")
    def hello():
        return "Hello!"
    ```
  </Tab>
  <Tab title="Instrument globally">
    ```python
    from opentelemetry.instrumentation.flask import FlaskInstrumentor

    # Patches flask.Flask so all future app instances are instrumented
    FlaskInstrumentor().instrument()

    from flask import Flask
    app = Flask(__name__)
    ```
  </Tab>
</Tabs>

## Configuration

<ParamField path="request_hook" type="callable">
  Called right after a span is created for an incoming request. Receives `(span, environ)` where `environ` is a `WSGIEnvironment`.
</ParamField>

<ParamField path="response_hook" type="callable">
  Called right before the span is finished. Receives `(span, status, response_headers)`.
</ParamField>

<ParamField path="excluded_urls" type="string">
  Comma-delimited list of regexes matching URLs to exclude from tracing. Overrides the `OTEL_PYTHON_FLASK_EXCLUDED_URLS` environment variable.
</ParamField>

<ParamField path="enable_commenter" type="boolean" default="True">
  Enables SQLCommenter support, which appends OpenTelemetry trace context to outgoing SQL queries.
</ParamField>

<ParamField path="commenter_options" type="object">
  Dictionary to opt out of specific SQLCommenter key-value pairs. Supported keys: `framework`, `route`, `controller`. Set a key to `False` to exclude it.
</ParamField>

<ParamField path="tracer_provider" type="TracerProvider">
  Optional tracer provider to use. Defaults to the globally configured provider.
</ParamField>

<ParamField path="meter_provider" type="MeterProvider">
  Optional meter provider to use. Defaults to the globally configured provider.
</ParamField>

### Environment variables

| Variable | Description |
|---|---|
| `OTEL_PYTHON_FLASK_EXCLUDED_URLS` | Comma-delimited URL regex exclusion list. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_REQUEST` | Request header names (or regexes) to capture as span attributes. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_RESPONSE` | Response header names (or regexes) to capture as span attributes. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SANITIZE_FIELDS` | Header names whose values should be replaced with `[REDACTED]`. |

### Hooks example

```python
from opentelemetry.trace import Span
from wsgiref.types import WSGIEnvironment
from typing import List
from opentelemetry.instrumentation.flask import FlaskInstrumentor

def request_hook(span: Span, environ: WSGIEnvironment):
    if span and span.is_recording():
        span.set_attribute("custom.attribute", "value")

def response_hook(span: Span, status: str, response_headers: List):
    if span and span.is_recording():
        span.set_attribute("custom.response_attribute", "value")

FlaskInstrumentor().instrument(
    request_hook=request_hook,
    response_hook=response_hook,
)
```

## Spans produced

A single server span is created per HTTP request with kind `SERVER`. The span name is set to the matched URL rule pattern, e.g. `GET /users/<int:id>`. Standard attributes include:

- `http.method` / `http.request.method`
- `http.route`
- `http.status_code` / `http.response.status_code`
- `http.scheme`
- `http.host`
- `http.target`
- `http.user_agent`

## Supported versions

Requires `flask >= 1.0`.
