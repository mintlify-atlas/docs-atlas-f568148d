<Warning>
This package implements GenAI semantic conventions that are currently in **development** status. The API surface, span attributes, and metric names may change in future releases as the conventions evolve.
</Warning>

## Installation

```bash
pip install opentelemetry-instrumentation-langchain
```

## Usage

### Auto-instrumentation

Use the `opentelemetry-instrument` CLI to instrument without code changes:

```bash
opentelemetry-instrument python app.py
```

### Manual instrumentation

Call `LangChainInstrumentor().instrument()` before invoking any LangChain chains or models. The instrumentor registers an `OpenTelemetryLangChainCallbackHandler` with the LangChain callback manager, which captures telemetry for all LLM interactions.

```python
from langchain_core.messages import HumanMessage, SystemMessage
from langchain_openai import ChatOpenAI
from opentelemetry.instrumentation.langchain import LangChainInstrumentor

LangChainInstrumentor().instrument()

llm = ChatOpenAI(model="gpt-3.5-turbo", temperature=0, max_tokens=1000)

messages = [
    SystemMessage(content="You are a helpful assistant!"),
    HumanMessage(content="What is the capital of France?"),
]

result = llm.invoke(messages)
print(result.content)
```

To stop instrumentation:

```python
LangChainInstrumentor().uninstrument()
```

<Note>
The instrumentation currently supports `ChatOpenAI` and `ChatBedrock` providers. Other providers may be supported in the future; telemetry for unsupported providers is silently skipped.
</Note>

## Configuration

<ParamField path="tracer_provider" type="TracerProvider">
  Custom `TracerProvider` to use. Defaults to the global tracer provider.
</ParamField>

<ParamField path="meter_provider" type="MeterProvider">
  Custom `MeterProvider` to use. Defaults to the global meter provider.
</ParamField>

<ParamField path="logger_provider" type="LoggerProvider">
  Custom `LoggerProvider` for emitting log events. Defaults to the global logger provider.
</ParamField>

### Environment variables

| Variable | Description |
|---|---|
| `OTEL_INSTRUMENTATION_GENAI_CAPTURE_MESSAGE_CONTENT` | Set to `true` to capture prompt and completion content as span or event attributes. Content is **not captured by default**. |

## Spans produced

`LangChainInstrumentor` produces spans for each LLM call initiated through the LangChain callback system. Span structure mirrors the GenAI semantic conventions.

| Attribute | Description |
|---|---|
| `gen_ai.system` | AI provider (e.g., `"openai"`, `"aws.bedrock"`) |
| `gen_ai.operation.name` | Operation type, e.g., `"chat"` |
| `gen_ai.request.model` | Model name requested |
| `gen_ai.response.model` | Model name returned in the response |
| `gen_ai.usage.input_tokens` | Number of prompt tokens consumed |
| `gen_ai.usage.output_tokens` | Number of completion tokens produced |

## Supported versions

Requires `langchain >= 0.3.21`.
