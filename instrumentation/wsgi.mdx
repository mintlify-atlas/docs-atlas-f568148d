---
title: "WSGI"
description: "OpenTelemetry instrumentation for WSGI — traces HTTP requests and responses automatically."
---

Provides a PEP 3333 conforming WSGI middleware (`OpenTelemetryMiddleware`) that can wrap any WSGI application to trace request timing through OpenTelemetry.

## Installation

```bash
pip install opentelemetry-instrumentation-wsgi
```

## Usage

### Auto-instrumentation

`opentelemetry-instrument` handles WSGI-based frameworks (Flask, Django) automatically.

### Manual instrumentation

Wrap your WSGI application callable with `OpenTelemetryMiddleware`.

<Tabs>
  <Tab title="Flask">
    ```python
    from flask import Flask
    from opentelemetry.instrumentation.wsgi import OpenTelemetryMiddleware

    app = Flask(__name__)
    app.wsgi_app = OpenTelemetryMiddleware(app.wsgi_app)

    @app.route("/")
    def hello():
        return "Hello!"
    ```
  </Tab>
  <Tab title="Django">
    ```python
    # wsgi.py
    import os
    from django.core.wsgi import get_wsgi_application
    from opentelemetry.instrumentation.wsgi import OpenTelemetryMiddleware

    os.environ.setdefault("DJANGO_SETTINGS_MODULE", "myproject.settings")

    application = get_wsgi_application()
    application = OpenTelemetryMiddleware(application)
    ```
  </Tab>
  <Tab title="Raw WSGI">
    ```python
    from opentelemetry.instrumentation.wsgi import OpenTelemetryMiddleware

    def my_app(environ, start_response):
        start_response("200 OK", [("Content-Type", "text/plain")])
        return [b"Hello, World!"]

    my_app = OpenTelemetryMiddleware(my_app)
    ```
  </Tab>
</Tabs>

## Configuration

`OpenTelemetryMiddleware` accepts the following constructor arguments:

<ParamField path="request_hook" type="callable">
  Called right after a span is created. Receives `(span, environ)` where `environ` is the `WSGIEnvironment`.
</ParamField>

<ParamField path="response_hook" type="callable">
  Called right before the span is finished. Receives `(span, environ, status, response_headers)`.
</ParamField>

<ParamField path="tracer_provider" type="TracerProvider">
  Optional tracer provider. Defaults to the globally configured provider.
</ParamField>

<ParamField path="meter_provider" type="MeterProvider">
  Optional meter provider. Defaults to the globally configured provider.
</ParamField>

### Environment variables

| Variable | Description |
|---|---|
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_REQUEST` | Request header names (or regexes) to capture as span attributes. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SERVER_RESPONSE` | Response header names (or regexes) to capture as span attributes. |
| `OTEL_INSTRUMENTATION_HTTP_CAPTURE_HEADERS_SANITIZE_FIELDS` | Header names whose values should be replaced with `[REDACTED]`. |
| `OTEL_PYTHON_INSTRUMENTATION_HTTP_CAPTURE_ALL_METHODS` | Set to `1` to record non-standard HTTP methods instead of labeling them `NONSTANDARD`. |

### Hooks example

```python
from opentelemetry.trace import Span
from wsgiref.types import WSGIEnvironment
from opentelemetry.instrumentation.wsgi import OpenTelemetryMiddleware

def request_hook(span: Span, environ: WSGIEnvironment):
    if span and span.is_recording():
        span.set_attribute("custom.attribute", environ.get("REMOTE_ADDR", ""))

def response_hook(span: Span, environ: WSGIEnvironment, status: str, response_headers: list):
    if span and span.is_recording():
        span.set_attribute("custom.status", status)

app = OpenTelemetryMiddleware(app, request_hook=request_hook, response_hook=response_hook)
```

## Spans produced

A single server span is created per HTTP request with kind `SERVER`. The span name is the HTTP method and URL path, e.g. `GET /api/users`. Standard attributes include:

- `http.method` / `http.request.method`
- `http.status_code` / `http.response.status_code`
- `http.scheme`
- `http.host`
- `http.target`
- `http.url`
- `http.user_agent`
- `http.flavor`

## Supported versions

No specific framework version requirement — works with any PEP 3333 compliant WSGI application.
